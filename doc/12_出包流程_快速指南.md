## 二、抖音小游戏出包

### SDK 状态
- ByteDance StarkSDK v6.7.7 → 本地包路径：`LocalPackages/com.bytedance.starksdk@6.7.7/`
- 抖音模板：`LocalPackages/com.bytedance.starksdk@6.7.7/DefaultTemplate/`

### 构建步骤

```
1. 团结编辑器菜单 → File → Build Profiles
2. 选择平台：ByteGame MiniGame（或 MiniGame → 切换子平台为抖音）
3. 配置 Player Settings：
   - Product Name: 铲屎官疯了
   - 竖屏 (Portrait)
   - WebGL Memory Size: 256MB
4. 点击 Build → 选择输出目录
5. 生成 .tt 包
```

### 真机测试

```
1. 安装「抖音开发者工具」
   下载：https://developer.open-douyin.com/docs/resource/zh-CN/mini-game/develop/guide/game-engine/rd-to-SCgame/BGDT-handbook/install
2. 打开 .tt 包 → 扫码预览
3. 真机调试 → 查看 Console / 性能数据
```

### 性能指标

| 指标 | 目标 | 上限 |
|------|------|------|
| 首帧时间 | < 3s | < 5s |
| 首包大小 | < 2MB | < 4MB |
| 内存占用 | < 200MB | < 400MB |
| FPS | 60 | ≥ 30 |

### 常见问题

| 问题 | 解决 |
|------|------|
| 包太大 | 打开 Compress Texture / Sprite Atlas / 图集压缩 |
| 首帧慢 | 减小首包、懒加载资源、Addressables 分包 |
| 闪退 | 检查 IL2CPP Stripping、添加 link.xml |
| 抖音 API 调用失败 | 检查 TTSDK.jslib 是否正确加载 |

## 三、微信小游戏出包

### SDK 状态
- WX-WASM-SDK-V2 v0.1.32 → `Assets/WX-WASM-SDK-V2/`
- 配置档：`Assets/WX-WASM-SDK-V2/Editor/MiniGameConfig.asset`

### 配置 MiniGameConfig（必须）

在 Unity 中选中 `Assets/WX-WASM-SDK-V2/Editor/MiniGameConfig.asset`：

```
ProjectConf:
  projectName: 铲屎官疯了
  Appid: wx________________      ← 微信小游戏 AppID（必填）
  CDN: https://cdn.xxx.com/      ← （可选）CDN 资源地址
  MemorySize: 256
  Orientation: 0                   ← 竖屏
  assetLoadType: 0                 ← 小包加载
```

### 构建步骤

```
1. 团结编辑器菜单 → File → Build Profiles
2. 选择平台：WeixinMiniGame（子平台 8）
3. 或使用：微信小游戏 → 导出 WebGL
4. 点击 Build
5. 输出目录 → 微信开发者工具导入
```

### 真机测试

```
1. 安装「微信开发者工具」
   下载：https://developers.weixin.qq.com/minigame/dev/devtools/download.html
2. 导入构建输出目录
3. 预览 → 扫码 → 真机调试
```

## 四、快速迭代建议

### 开发阶段 → 先跑 WebGL 本地预览

团结引擎支持直接 Build 为 WebGL，在浏览器里快速预览：

```
File → Build Profiles → WebGL → Build and Run
```

- 优点：不需要每次上传到抖音/微信开发者工具
- 注意：部分平台 API（广告、支付）在纯 WebGL 中不可用

### 平台切换

团结引擎的 MiniGame 构建目标支持子平台切换：

```
Player Settings → MiniGame → 子平台：抖音 / 微信
```

或直接在 Build Profiles 中选择对应的 Build Target。

## 五、F8Framework 与出包

F8Framework 通过 GitHub 源引用，Unity 构建时会自动包含：
- FSM 状态机
- 资源管理
- UI 系统

**无需额外配置**。F8Framework Editor 目录下的 `F8EditorConfig.asset` 仅编辑器用，不会被打包。
