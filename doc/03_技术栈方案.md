## 2. 框架模块使用计划

### 2.1 F8Framework 模块矩阵

| 模块 | 用途 | 优先级 |
|------|------|--------|
| F8.Framework | 框架核心启动 | P0 |
| UI 管理 | UI 面板生命周期、层级管理 | P0 |
| 音频管理 | BGM/SFX 播放控制 | P0 |
| 事件系统 | 模块解耦通信 | P0 |
| 对象池 | UI 元素复用、粒子效果 | P0 |
| 有限状态机 (FSM) | 游戏流程状态管理 | P0 |
| 输入管理 | 点击/触摸事件 | P1 |
| 资源管理 | Addressables 集成 | P1 |
| 设置管理 | 玩家设置持久化 | P1 |
| 本地化 | 多语言支持 | P2 |
| 网络请求 | HTTP/WebSocket | P2 |
| SDK 管理 | 广告 SDK 集成 | P1 |
| 计时器 | 倒计时/延迟调用 | P0 |
| 协程管理 | 异步流程控制 | P1 |

### 2.2 不使用的模块

| 模块 | 原因 |
|------|------|
| 热更新 (HybridCLR) | 初期不需要，体量小时直接发版 |
| Tween 动画 | 优先使用 Unity 原生 DOTween 或 UGUI 动画 |
| 寻路 (NavMesh) | 2D 小游戏无需 |
| 物理引擎高级 | 仅使用 2D 碰撞检测基本功能 |

## 5. 性能目标

### 5.1 抖音小游戏平台限制

| 指标 | 限制值 | 目标值 |
|------|--------|--------|
| 首包大小 | < 4MB | < 2MB |
| 总包大小 | < 20MB (分包) | < 15MB |
| 首帧时间 | < 5s | < 3s |
| 运行内存 | IOS < 400MB, Android 依机型 | < 200MB |
| FPS | 60 | 稳定 60 |
| DrawCall | - | < 30 |

### 5.2 性能优化策略

| 策略 | 说明 |
|------|------|
| Sprite Atlas | 所有 UI 和关卡图打包 Atlas，减少 DrawCall |
| 对象池 | UI 元素和常见特效预创建，避免频繁 Instantiate |
| 资源异步加载 | Addressables 或 F8 资源模块异步加载关卡图 |
| 图集压缩 | PNG 压缩 + WebGL 纹理格式 ETC2/ASTC |
| 代码剥离 | Player Settings → Strip Engine Code |
| 字体裁剪 | TextMeshPro 字体 Atlas 只包含使用字符 |
