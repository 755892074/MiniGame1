## 2. 日常开发循环

### 2.1 每日启动

```
1. 打开团结编辑器，打开项目
2. 确保 Codely MCP 桥接运行中（编辑器自动启动）
3. 在 WorkBuddy 中开始工作
```

### 2.2 WorkBuddy → 团结编辑器（Codely）

| 操作 | 工具 | 示例 |
|------|------|------|
| 创建脚本 | unity_script | "创建 FindDiffController.cs" |
| 创建场景 | unity_scene | "创建 Main.scene" |
| 搭建UI | unity_gameobject | "创建 Canvas → 添加按钮" |
| 执行代码 | execute_csharp_script | "批量生成 10 个关卡 SO" |
| 截图预览 | unity_screenshot | "截取当前场景" |
| 编辑器操作 | unity_editor | "检查编辑器状态" |

### 2.3 美术资产（手动流程）

```
WorkBuddy 输出提示词 → 你复制到 Image2 → 生成 → 下载 → 
导入到 Assets/Art/ → WorkBuddy 通过 unity_asset 配置 Sprite 设置
```

## 3. 典型开发场景

### 场景1：创建新关卡

```
你：在 WorkBuddy 中 say "创建第5关找茬关卡"
WorkBuddy：
  1. execute_csharp_script 创建 LevelData SO
  2. 生成 Image2 提示词 → 你手动生成
  3. 你导入图片后，unity_asset 配置 Sprite
  4. 设置差异点坐标数据
```

### 场景2：创建新 UI 面板

```
你：在 WorkBuddy 中 say "创建设置面板"
WorkBuddy：
  1. unity_gameobject create Canvas → 添加 Panel → SettingPanel
  2. unity_gameobject create 按钮 → 音量/反馈/关于
  3. unity_script 创建 SettingsPanel.cs
  4. unity_gameobject set_component_property 绑定引用
  5. unity_screenshot 截图预览
```

### 场景3：批量操作

```
你：在 WorkBuddy 中 say "给所有关卡SO设置timeLimit=60"
WorkBuddy：
  execute_csharp_script 遍历 Assets 目录下所有 LevelData，
  批量修改 timeLimit 字段
```

### 场景4：Bug 修复

```
你：在 WorkBuddy 中 say "修复找茬点击判定的边界问题"
WorkBuddy：
  1. unity_script 读取 DifferenceManager.cs
  2. 分析代码，找到问题
  3. unity_script 修改代码
  4. unity_console 检查无新错误
  5. unity_screenshot 截取游戏画面确认修复
```

## 4. 提示词工作流

### 4.1 美术提示词流程

```
1. WorkBuddy → 根据关卡主题生成 Image2 提示词
2. 你 → 复制到 Image2 网页 → 生成 → 下载
3. WorkBuddy → 通过 unity_asset 导入并配置
4. 重复 → 生成对比图（同 prompt + 差异指令）
```

### 4.2 视频提示词流程

```
1. WorkBuddy → 根据过场需求生成 SeedDance 提示词
2. 你 → 复制到 SeedDance 2.0 → 生成 → 下载
3. WorkBuddy → unity_asset 导入视频到项目
```

---

## 5. 版本管理集成

每天工作结束：

```
1. WorkBuddy → unity_scene save 保存当前场景
2. 终端 → git status 查看变更
3. 终端 → git add + git commit（遵循 Conventional Commits）
4. 终端 → git push 推送到远程
```
