> 版本：v1.0 | 日期：2026-07-19

## 一、平台包体限制（抖音小游戏，官方）

来源：抖音开放平台「分包加载」「代码包体积限制」。

| 场景 | 整体包上限 | 主包 | 单分包 |
|------|-----------|------|--------|
| 未分包 | 20MB | — | — |
| 分包（默认） | 20MB | **4MB** | 20MB |
| 开通虚拟支付后 | 30MB | 4MB | 30MB |

- 分包基础库要求 **1.88.0+**（`tt.loadSubpackage(res)` 返回 `LoadSubpackageTask`，可监听 `progress`）。
- 超过主包/分包限制的资源 **必须走网络下载**（CDN / 分包）。
- 允许上传的文件类型含：`png/jpg/ogg/mp3/ttf/otf/prefab/scene/fbx/astc/glb/...`（美术/音频/字体/预制体均合法）。

**结论**：首包（主包）必须压在 **4MB 内**，且整体 ≤20MB。我们的代码/WASM/首场景 + MVP 美术要塞进这 4MB。

## 二、团结引擎（Tuanjie）资源方案对比

当前工程为 **Tuanjie 2022.3.62t11**。团结引擎针对小游戏有专门能力，方案选型如下：

| 方案 | 原理 | 优点 | 缺点 | 适用 |
|------|------|------|------|------|
| **AutoStreaming + Instant Game** | 引擎自动把重资源（Texture/Audio/Mesh/Animation/Scene/Font）拆出首包，运行时按 CDN 懒加载；打原图替换为 32×32 占位图，下载后自动替换 | **零代码改动**；自动双纹理格式(ASTC+DXT)按设备切换；原生支持 | 会看到加载过程，需 loading 遮挡；首场景不能开 Streaming（黑屏） | 首推：美术/音频等"重资源"自动远程化 |
| **Addressables** | Unity 官方资源框架，按 Label/Group 分包，Remote.LoadPath 指向 CDN，自动管理下载/缓存/版本 | 逻辑分包清晰（关卡/宠物/建筑按需加载）；官方维护；可配合 AutoStreaming | 接口异步，大量同步加载改造成本高；索引文件偏大 | 关卡/宠物/建筑等**逻辑内容**按需加载 |
| YooAsset | 国产开源框架 | 简单易用 | Package 间资源不去重，包体变大 | 不推荐（与 AutoStreaming/Addressables 重叠） |
| 原生 AssetBundle | 手写框架 | 完全可控 | 开发量大 | 仅特殊需求 |

**推荐组合**：`AutoStreaming`（重美术/音频自动远程）+ `Addressables`（关卡/宠物/建筑逻辑分包）。两者都上传到 **UOS CDN**（团结引擎云服务），版本管理一体化。

## 三、推荐架构：首包 + 远程 两层

```
首包（主包 ≤4MB，随包下发，Player 启动即读）
 ├─ 代码 / WASM / IL2CPP
 ├─ 首场景（PetGameScene / MenuScene）基础依赖
 ├─ MVP 美术（设计 §8.1：约 9 张，<200KB）
 │    ├─ 货币图标 ×3（金币/鱼干/徽章）
 │    ├─ 建筑图标 ×4（食盆/玩具/医疗/花园）
 │    ├─ 结算面板背景 ×1
 │    └─ 爱心表情 ×1（喂食反馈）
 └─ 占位图（AutoStreaming 自动生成，极小）

远程（UOS CDN，按需下载，缓存到本地）
 ├─ 后续美术（设计 §8.2）
 │    ├─ 变异换色贴图 ×6
 │    ├─ 专属小窝外观 ×6
 │    ├─ 粒子特效贴图 2-3
 │    └─ 成长阶段外观 ×12（少年/成年）
 ├─ 后续关卡资源（Addressables 按关卡 Group）
 └─ 新增宠物/建筑（Addressables Label）
```

## 四、项目目录规划（配置驱动，已就绪）

当前美术在 `Assets/Art/`（6 宠物×7 表情、15 食物、8 饭碗、8 UI）。建议拆分：

| 目录 | 内容 | 加载方式 |
|------|------|----------|
| `Assets/Art/MVP/` | 首包 MVP 美术（上述 9 张） | 随包 / AutoStreaming 占位 |

## 五、Addressables 远程加载配置

### 5.1 Profile 设置

- 在 `Addressables Profiles` 新建 `Douyin` Profile：
  - `Remote.LoadPath` = `https://a.unity.cn/client_api/v1/buckets/<BUCKET_ID>/release_by_badge/latest/content/`
  - `Remote.BuildPath` = `ServerData`
- 构建时选择 `Douyin` Profile；**首包资源 Group 的 `Bundle Mode` 选 `Pack Together`（本地）**，远程 Group 选 `Remote.BuildPath`。

### 5.2 ⚠️ dummy 占位符修复（抖音必踩）

抖音小游戏下 `Addressables` 的 `Remote.LoadPath` 常出现 `https://dummy.dummy.dummy/` 占位符且不被自动接管，导致资源 404。修复：在工程中放一个 **无需挂载** 的 `[Preserve]` 脚本，用 `Addressables.InternalIdTransformFunc` 把 dummy 域名替换为真实 CDN 地址（源自团结引擎踩坑博客）：

```csharp
// Assets/EditorOrRuntime/SkipUnityLogo.cs  —— 放工程 Assets 目录下即可，无需挂载
#if !UNITY_EDITOR
using UnityEngine;
using UnityEngine.AddressableAssets;
using UnityEngine.ResourceManagement.ResourceLocations;

[Preserve]
public class FixDouyinAddressablePath
{
    [RuntimeInitializeOnLoadMethod(RuntimeInitializeLoadType.BeforeSceneLoad)]
    private static void Fix()
    {
        // 把 dummy 占位符替换为 UOS CDN 真实地址（结尾要带 /）
        const string realCdn = "https://a.unity.cn/client_api/v1/buckets/<BUCKET_ID>/release_by_badge/latest/content/";
        Addressables.InternalIdTransformFunc = location =>
        {
            string id = location.InternalId;
            if (id.StartsWith("https://dummy.dummy.dummy/"))
                id = id.Replace("https://dummy.dummy.dummy/", realCdn);
            if (id.Contains("StreamingAssets/StreamingAssets/"))
                id = id.Replace("StreamingAssets/StreamingAssets/", "StreamingAssets/");
            return id;
        };
    }
}
#endif
```

> 若 CDN 返回 404，多半是 `release_by_badge/latest` 没做版本发布或 CLI 上传路径含空格。`uas releases create` 做版本管理后通常解决。

## 六、UOS CDN 上传流程

1. 团结引擎后台开启 **UOS CDN**（Instant Game 打包时 Copy 对应 CDN 地址，绑定 UOS 存储桶）。
2. 下载 **uas CLI**（Windows 版），命令行上传：
   ```bat
   cd /d C:\path\to\uas_win_V1.0.9
   uas.exe
   uas auth login --uos_app_id <APPID> --uos_app_service_secret <SECRET>
   :: respath = build 结束后的 webgl 输出目录（路径不可含空格）
   uas entries sync --bucket <BUCKET_ID> <RES_PATH>
   :: 版本管理（否则拉到旧资源）
   uas releases create --bucket <BUCKET_ID> --notes "LatestUpload"
   ```
3. 抖音开发者工具中填 **链接白名单**（注意：去掉 `https://` 前缀）。

## 七、构建与发布步骤（抖音小游戏）

1. 团结引擎切换到 **Minigame 模式**，再切到 **抖音开发环境**（默认 TTSDK，无需手切）。
2. 开启 **Instant Game（AutoStreaming）** + 配置 `Douyin` Addressables Profile。
3. 填 **Douyin Profile**（appid、CDN 地址、首包前缀）。
4. 直接 **Build**（抖音可上传内容输出到 `ByteGameOutput`）。
5. `uas` 上传资源到 UOS CDN（见第六节）。
6. 抖音开发者工具导入 `ByteGameOutput` → 预览/真机测试 → 提交审核。

> 传统 WebGL→转换流程（先构建 WebGL 再转）在抖音上容易报错，建议走 Minigame 模式直接 Build。

## 八、关键坑与风险

| 坑 | 说明 | 对策 |
|----|------|------|
| 首场景开 Streaming 黑屏 | 首场景不要开启 AutoStreaming | 首场景资源随包；非首场景才走 Streaming |
| 共享 AB 过大 | 共享资源 AB 控制在 **3MB 内**，别打 FBX 进去 | 拆分共享包 |
| 主包选 CDN 打包中断 | 首包资源只能选 `Package`，不能选 CDN | 首包资源本地打包 |
| dummy 占位符 404 | 抖音不自动接管 Addressables 远程路径 | 用第五节脚本替换 |
| 链接白名单 | 抖音需填白名单且去 `https://` | 构建后配置 |
| WASM 体积 | 首包大头是 WASM，编译内存≈体积×10 | Extreme 托管代码精简 + 引擎代码剔除 + 不用 `Resources/` 文件夹 |
| 版本未对齐 | 旧资源被缓存 | `uas releases create` 做版本管理 |
