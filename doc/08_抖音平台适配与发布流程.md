# 08 抖音平台适配与发布流程

---

## 1. 平台适配概览

```
Unity/Tuanjie 项目
        │
        ↓ ByteGame MiniGame 平台切换
        │
抖音小游戏包 (.tt)
        │
        ↓ 抖音开发者工具上传
        │
抖音小游戏审核
        │
        ↓ 通过
        │
正式上线
```

---

## 2. TTSDK 集成步骤

### 2.1 环境准备

1. 下载 TTSDK：https://developer.open-douyin.com/docs/resource/zh-CN/mini-game/develop/guide/game-engine/rd-to-SCgame/BGDT-handbook/install
2. 导入 `.unitypackage` 到项目（全选导入）
3. File → Build Settings → 选择 **ByteGame MiniGame** → Switch Platform
4. 菜单栏 ByteGame → Platform Plugin List → 启用 WebGL Support + ByteGame MiniGame Support

### 2.2 Player Settings 关键配置

| 设置项 | 值 | 说明 |
|--------|-----|------|
| Company Name | 你的公司名 | 拼音或英文 |
| Product Name | 找茬大师 | 游戏名称 |
| Default Icon | 512×512 PNG | 游戏图标 |
| Run In Background | ✓ | 避免切后台出错 |
| Color Space | Linear | 颜色更准确 |
| Auto Graphics API | ✗ | 仅保留 WebGL 2.0 |
| Strip Engine Code | ✓ | 减小包体 |
| Scripting Backend | IL2CPP | 性能更好 |

### 2.3 微信/抖音双平台共用

Tuanjie 支持一键切换 WebGL → ByteGame MiniGame 平台。

## 3. 广告SDK接入

### 3.1 广告位创建

在抖音开放平台控制台创建广告位，获取广告位 ID：

| 广告位 | 类型 | ID 示例 |
|--------|------|---------|
| 激励视频-翻倍 | rewardVideo | `adunit-xxx-01` |
| 激励视频-复活 | rewardVideo | `adunit-xxx-02` |
| 激励视频-道具 | rewardVideo | `adunit-xxx-03` |
| 插屏广告 | interstitial | `adunit-xxx-04` |
| Banner | banner | `adunit-xxx-05` |

### 3.2 广告SDK初始化

```csharp
// AdManager.cs - 初始化
void Start() {
    // 初始化广告SDK
    TT.Init();
    
    // 预加载广告
    PreloadRewardVideo("adunit-xxx-01");
    PreloadInterstitial("adunit-xxx-04");
}

// 展示激励视频
public void ShowRewardVideo(string adUnitId, Action onReward) {
    TT.ShowRewardVideo(adUnitId, (result) => {
        if (result == TTRewardVideoResult.Completed) {
            onReward?.Invoke();
        }
    });
}
```

## 4. 打包构建

### 4.1 构建前检查

| 检查项 | 操作 |
|--------|------|
| Sprite Atlas 已设置 | Assets → Create → Sprite Atlas |
| 资源引用完整 | Codely 运行完整性检查 |
| UI 分辨率适配 | Canvas Scaler 设置为 Scale With Screen Size |
| 字体已裁剪 | TextMeshPro 字体 Atlas 只含使用字符 |

```
File → Build Settings → ByteGame MiniGame → Build
→ 选择输出目录
→ 生成 .tt 包
```

## 5. 提审流程

### 5.1 前置条件

| 条件 | 说明 |
|------|------|
| 软著证书 | 必须，申请周期 30-60 天 |
| 抖音开发者账号 | 已实名认证 |
| 游戏名称已确认 | 不超过 8 个汉字 |
| 游戏图标 | 512×512 圆角 PNG |
| 游戏截图 | 至少 3 张，1080×1920 |
| 隐私政策 | 必须，即使不收集数据 |

### 5.2 提审步骤

1. 登录 [抖音开放平台](https://developer.open-douyin.com/)
2. 创建小游戏 → 填写基本信息
3. 上传软件包 (.tt)
4. 上传游戏截图和图标
5. 配置广告位
6. 填写隐私政策
7. 提交审核（审核周期 1-3 个工作日）

### 5.3 常见拒审原因

| 原因 | 预防措施 |
|------|---------|
| 缺少隐私政策 | 必须加入隐私政策弹窗 |
| 诱导分享 | 不强制分享，分享奖励合理 |
| 广告违规 | 广告位符合规范，不遮挡游戏内容 |
| 内容违规 | 无暴力/色情/政治内容 |
| 性能不达标 | 首帧 < 5s，内存 < 400MB |

## 6. 运营工具

### 6.1 抖音开发者工具

- 下载：抖音开放平台 → 开发工具
- 功能：预览调试、真机测试、性能分析、日志查看

### 6.2 数据看板

抖音开放平台提供：
- 用户数据（DAU、新增、留存）
- 广告数据（展示量、eCPM、收入）
- 性能数据（加载时间、崩溃率）

## 7. 更新发版流程

```
1. 修改 Application.version
2. 打包新 .tt 包
3. 上传到抖音开放平台
4. 填写更新日志
5. 提交审核
6. 审核通过 → 自动更新
```
