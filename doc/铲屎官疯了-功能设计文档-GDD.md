## 一、游戏概述

### 1.1 一句话描述

> 一群宠物排队等开饭，你把混在一起的食物按顺序分给它们——分错了猫炸毛、分对了狗摇尾。

### 1.2 核心玩法

```
食物堆(栈) → 取顶部食物 → 喂给队首宠物
                        → 不匹配？→ 暂存碗中转
                        → 匹配 ✓ → 宠物吃一口
                        → 碗满了 → 宠物开心离开 → 下一个上前
```

### 1.3 差异化亮点

| 竞品 | 我们 |
|------|------|
| 开个大脑门（倒水排序）| 排队分食（有故事有表情）|
| 羊了个羊（故意卡死）| 算法保证可解（逆向推导）|
| 传统找茬 | 宠物表情包自带传播 |

## 二、游戏规则

### 2.1 六条核心规则

```
规则1: 只能从食物堆顶部取食物（栈顶）
规则2: 只能喂给当前排在第一位的宠物
规则3: 食物类型必须匹配宠物类型（猫粮→猫，狗粮→狗...）
规则4: 宠物碗有容量上限（默认4层）
规则5: 碗满了 → 宠物吃饱离开 → 下一只上前
规则6: 不匹配的食物 → 放入暂存碗中转 → 可倒回食物堆改变顺序
```

### 2.2 胜利条件

> 所有宠物的碗都装满 → 全部吃饱 → 通关 🎉

### 2.3 失败条件

| 条件 | 触发 |
|------|------|
| 时间耗尽 | 关卡时限到达 |
| 错误上限 | 喂错N次（可选规则，默认关闭）|

## 五、交互流程

### 5.1 一次完整的喂食操作

```
1. 玩家点击食物堆 → 取出顶部食物（到"手上"）
   ↓
2. 玩家点击队首宠物的碗
   ├─ 食物匹配 ✓ → 食物入碗 → 宠物开心表情
   │   ├─ 碗还没满 → 返回步骤1
   │   └─ 碗满了 → 宠物😇离开 → 下一只上前
   └─ 食物不匹配 ✗ → 宠物🤮痛苦表情 → 自动放入暂存碗(0号)
   ↓
3. 玩家点击暂存碗 → 取出暂存的食物
   ├─ 喂给队首宠物（步骤2）
   ├─ 放入另一个暂存碗
   └─ 倒回食物堆（改变栈顶顺序）
   ↓
4. 重复直到通关
```

## 六、关卡设计

### 6.1 难度分级

| 难度 | 关卡 | 宠物数 | 暂存碗 | 碗容量 | 典型步数 | 通关率 |
|------|------|--------|--------|--------|---------|--------|
| ⭐ 入门 | 1-10 | 2 | 2 | 4 | 6-12 | 85% |
| ⭐⭐ 普通 | 11-30 | 3 | 2 | 4 | 12-25 | 65% |
| ⭐⭐⭐ 困难 | 31-60 | 4 | 1 | 4 | 25-45 | 40% |
| ⭐⭐⭐⭐ 地狱 | 61-100 | 5 | 1 | 4 | 45-80 | 15% |

### 6.2 关卡示例

**第3关：三宠混战**

```
宠物排队：🐹仓鼠 → 🐶狗 → 🐱猫
食物堆(顶→底)：[瓜子][骨头][猫条][狗粮][骨头][瓜子][猫条][狗粮]
暂存碗：2个
解法：取瓜子→喂仓鼠✓；取骨头→暂存碗1；取猫条→暂存碗2；
      取狗粮→暂存碗1已满；倒回骨头→暂存碗2已满→倒回猫条；
      取狗粮→暂存碗2；取骨头→喂狗✓...（约15步）
```

### 6.3 关卡生成算法（逆向推导）

```
1. 随机选择N只宠物，确定排队顺序
2. 为每只宠物分配正确食物（装满碗 = 完美状态）
3. 从完美状态出发，随机打乱食物堆顺序
4. 验证：DSF回溯求解 → 确认有解 → 输出关卡
5. 计算最优步数 → 设置星级阈值
```
