# 17 自动化验收框架 & 卡死规避方法论（铲屎官疯了 v2）

> 沉淀自 2026-07 多轮 codely-unity 桥接实测。目标：让 AI 能"自己跑通关、下一关、模拟看广告、给 UI 做体检"，且不再因桥接卡死而半途崩。

---

## 一、为什么 codely 桥接会"卡死"（今晚实踩的 7 个坑）

| # | 现象 | 根因 | 卡死点 |
|---|------|------|--------|
| 1 | 点「提示」/「GM」后桥接断连、回传无日志 | `PetGameSolver` BFS 同步跑（上限 6 万状态）阻塞 Unity 主线程，桥接回传超时 | 主线程阻塞 |
| 2 | 连点两次「提示」，高亮被误判"叠加" | `UnityEngine.Object.Destroy` **延迟到帧末**执行；同段同步脚本里连点，旧 HL 还没销毁就被计数 | 帧时序 |
| 3 | 结算面板只剩黑底 / 碗会重排 | 预制体挂了 `LayoutGroup`（ResultOverlay=Vertical、BowlArea=Grid），覆盖代码里的 `anchoredPosition`/绝对定位 | prefab 配置 |
| 4 | 没操作就看到「不公平」气泡 | `EditorAutoDemo` 开发期强制 `lastFedIsFirst=false` 弹气泡，干扰验收 | 测试脚本副作用 |
| 5 | 脚本里 `FindObjectsOfType` 全场景遍历后回传失败 | 脚本执行重 + 回传采集时序竞争 | 回传超时 |
| 6 | Play 后立刻取状态/截图拿到的是 Menu 而非关卡 | `Bootstrap→MenuScene→LoadGame` 需要好几帧 | 时序等待 |
| 7 | 直接改 `isCompleted` 字段不刷新 UI | 绕过了 `onClick`/`onSelectionChanged` 真实路径 | 模拟路径错误 |

## 二、规避套路（写进肌肉记忆）

1. **断言走"写文件再 Read"，绝不在重脚本里用 `capture_logs:true`**。
   重逻辑（BFS、全场景遍历）后桥接极易断连，把结果 `File.WriteAllText("C:/Users/Administrator/WorkBuddy/Claw/temp_xx.txt", ...)` 再 `Read`。
2. **跨帧才连点**：涉及 `Destroy`/重建的连续操作，分两次 `execute_csharp_script` 调用，中间 `sleep 3` 让编辑器跑帧。
3. **运行时关掉布局组**：`FindRefs()` 里 `bowlArea.GetComponent<GridLayoutGroup>().enabled=false`；`OnWin()` 清空 `ResultOverlay` 后关掉其 `LayoutGroup`。prefab 上要么别挂、要么代码关。
4. **删 `EditorAutoDemo`**：开发期自动演示会伪造 UI 行为，验收必须靠脚本驱动真实交互。
5. **重 BFS 加护栏**：求解器限状态数（已有 6 万上限）+ 加协程分步或超时兜底，避免主线程卡死。
6. **时序等待**：`play` 后 `sleep 6~7s`（等 Menu 出现）；`LoadGame(n)` 后 `sleep 5s`（等 UI 构建）。
7. **走真实路径**：用 `btn.onClick.Invoke()` 或公开方法（如 `BuildBowls` 反射调用），不要直接改私有字段模拟状态。

## 四、自验收闭环设计（最终目标）

```
进游戏(Menu) → LoadGame(n) → 求解器算步骤 → 脚本逐步 onClick 通关
   → 结算面板体检(标题/星级/奖励/按钮可见)
   → 点「下一关」→ 验证关卡递增
   → 故意失败/点道具→触发广告弹窗→模拟"看完"→验证奖励发放
   → UI 体检: 元素拉伸/错位/锚点/文本零宽/用错图
   → 每步截图 + 写文件断言
```

**UI 体检项（自动可断言）**：
- 文本 `RectTransform.rect.width == 0` → 零宽不可见（如之前 GM 空白）
- 子物体被 `LayoutGroup` 接管导致 `sizeDelta` 异常
- 图片 `sprite == null` 或尺寸被拉伸（`rect` 比例 ≠ 原图比例）
- 按钮 `onClick` 持久事件数为 0
- 关键面板 `activeSelf` 但子节点数为 0（只剩背景）

## 五、安全边界

- **跑测卡死即停**：桥接断连 / 连续 2 次回传失败 → 停止当前测试，报告已验证部分，不强求跑完。
- **编译优先于运行**：每次改代码先 `request_compile` 确认 0 错误，再进 Play。
- **重逻辑不堆在一个脚本**：BFS、遍历、销毁分开调用，单次脚本只做一件事并写文件。
