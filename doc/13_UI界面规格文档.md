## 一、全局规范

### 1.1 设计尺寸

| 属性 | 值 |
|------|-----|
| 分辨率 | 750 × 1334（竖屏） |
| Canvas | ScreenSpaceOverlay |
| Scaler | ScaleWithScreenSize, match=1.0 |
| 字体 | LegacyRuntime.ttf（占位），后续替换为商用字体 |

### 1.2 配色方案

| 用途 | 色值 | 说明 |
|------|------|------|
| 主背景 | #F5F0E8 | 暖米白，温馨基调 |
| 顶栏/底栏 | #EFEBE3 | 略深米白 |
| 主按钮 | #F0997B → #D85A30 | 珊瑚橙渐变（暂时用纯色 #E47A52） |
| 次按钮 | #7F77DD → #534AB7 | 紫色系 |
| 文字-深 | #2C2C2A | 正文 |
| 文字-中 | #888780 | 副文本 |
| 文字-金 | #EF9F27 | 货币/高亮 |
| 文字-白 | #FFFFFF | 按钮上文字 |
| 背景遮罩 | rgba(0,0,0,0.6) | 弹窗背景 |

### 1.3 命名规范

```
Prefab 命名：PascalCase（如 MainMenuPanel, LoginPanel）
子物体命名：功能名+类型（如 btnStart, txtLevel, imgBg）
   btn  = Button
   txt  = Text
   img  = Image
   pnl  = Panel
   sld  = Slider
   tg   = Toggle
```

### 3.4 交互逻辑

| 状态 | 行为 |
|------|------|
| 首次启动 | `pnlPrivacyPopup` 显示，需点"同意"才能继续 |
| 点击"同意" | 隐藏弹窗，`SaveSystem.Data` 标记 `privacyAgreed=true` |
| 点击"不同意" | 弹窗保持，不可进入游戏 |
| 未勾选隐私 | 按钮 `interactable=false` |
| 抖音环境 | 隐藏 `btnDouyinLogin`，点击游客按钮直接 `tt.login()` |
| 非抖音 | 显示两个按钮，游客=本地存档，抖音=调起授权 |

## 七、场景架构

### 7.1 场景列表

| 场景 | Build Index | 说明 |
|------|-------------|------|
| BootScene | 0 | 启动场景，挂载 Bootstrap 脚本，初始化后跳转 |
| MenuScene | 1 | 主菜单场景（含 Splash/Login/MainMenu） |
| GameScene | 2 | 游戏场景（现有 PetGameScene 改名） |

### 7.2 启动流程

```
App 启动
  ↓
BootScene (index=0)
  ├── SaveSystem.Load()
  ├── 检测 SDK 环境
  └── SceneManager.LoadScene("MenuScene")
        ↓
MenuScene (index=1)
  ├── 第一次？ → 显示 LoginPanel
  │     ├── 首次 → 隐私弹窗 → 同意 → 进入
  │     └── 非首次 → 显示 LoginPanel（或直接 MainMenu）
  ├── 非第一次 → 直接显示 MainMenuPanel
  │
  ├── 点击"继续" → SceneManager.LoadScene("GameScene")
  ├── 点击"选关" → 弹出 LevelSelectPanel
  └── 点击"设置" → 弹出 SettingsPanel
        ↓
GameScene (index=2)
  ├── PetGameManager 初始化关卡
  ├── PetGameUI 加载 GameHUD
  ├── 通关 → 结算 → "回主菜单" → SceneManager.LoadScene("MenuScene")
  └── 通关 → "下一关" → 继续GameScene
```

### 7.3 SceneManager 脚本设计

    public static void LoadBoot() { ... }
