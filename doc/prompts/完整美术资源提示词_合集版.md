## 风格锚点（必须加到每张图的 Prompt 或 System Prompt）

**Positive 锚点（英文）**:
```
Studio Ghibli illustration style, Hayao Miyazaki aesthetic, hand-painted watercolor + oil painting blend,
natural warm colors, soft dappled sunlight filtering through trees (komorebi), lush organic textures,

no outlines or very subtle soft edges, rich but not oversaturated palette, warm golden hour lighting,

```

**Negative 锚点（英文，加到 Negative Prompt）**:
```
3D render, photorealistic, dark colors, gloomy, modern urban, sharp pixelated edges,
black outlines, thick outlines, cel shading, anime style, barbie style, neon, metallic,
high contrast, strong shadows, blur, text, watermark, signature, messy, chaotic
```

**关键风格规则**:
- **无描边或极淡描边**：吉卜力风格没有明显黑色粗线，靠色彩边缘自然区分
- **自然明亮色彩**：蓝天（非浅蓝，是深邃的吉卜力蓝）、草地（ lush green）、阳光（golden amber）
- **光影**：木漏れ日（树叶间洒下的阳光）、温暖侧光、柔和光晕
- **质感**：水彩+油画混合，能看到笔触纹理，但不粗糙
- **木质元素**：原木纹理、手工木制家具、温暖胡桃木/橡木色调
- **田园氛围**：野花、草地、篱笆、乡村小路、蓝天白云
- **氛围感**：微风拂动、阳光流转、治愈宁静

## 风格变体：动画友好型简化版

> 用于：宠物角色、表情、帧动画等需要游戏内动态使用的资源
> 核心原则：**保留吉卜力色彩和氛围，但大幅简化细节，确保帧与帧之间的一致性**

### 为什么需要简化版？

完整吉卜力风格（水彩+油画混合+复杂毛发笔触）在静态图上非常好看，但用于动画时会出现：
- **笔触抖动**：每帧AI生成的笔触方向不同，连续播放像在"抽搐"
- **细节漂移**：毛发纹理、光影位置帧间不一致
- **透明背景困难**：手绘边缘与背景融合，抠图后边缘不干净

### 简化版锚点（替代完整版，用于动画资源）

**Positive 锚点（英文）**:
```
Studio Ghibli inspired style, clean and simplified shapes, smooth color blocks with soft blending,
minimal fur detail, no complex individual hair strokes, clean edges suitable for game sprites,
cute big expressive eyes with clear pupils, soft dappled lighting without complex shadows,
warm natural colors, hand-painted feel but simplified for animation
```

**关键简化规则**:
- **减少毛发细节**：用大的色块表现毛色分区，不做单根毛发
- **保留吉卜力大眼特征**：大眼睛、高瞳孔、表情夸张（这是表情动画的核心）
- **平滑边缘**：角色与背景有明显色彩区分，方便抠图做透明背景
- **简化光影**：保留整体光照方向（顶光/侧光），不做复杂的环境反射
- **固定视角**：所有角色统一正面或3/4侧面，不做大透视
- **色块分明**：身体各部分（头/身/耳/尾）用清晰色块区分，方便后期做Spine骨骼分层

### 使用指南

| 资源类型 | 风格版本 | 原因 |
|----------|----------|------|
| Logo、背景、主视觉、UI面板 | **完整版** | 静态展示，细节越多越好 |
| 宠物表情、宠物idle动画 | **简化版** | 需要动画，帧间一致性优先 |
| 按钮、图标、道具 | 两者皆可 | 尺寸小，差异不明显 |
| 特效（水花、光斑）| **完整版** | 用粒子系统生成，不需要AI逐帧 |

## 快速选择指南

| 你的需求 | 选择版本 | 原因 |
|----------|----------|------|
| 做Logo、主视觉、背景图 | **完整版** | 细节丰富，视觉冲击力强 |
| 做UI按钮、面板、图标 | **完整版** | 尺寸小，细节差异不明显 |
| 做宠物表情切换动画 | **简化版** | 帧间一致性高，表情切换流畅 |
| 做idle呼吸/行走帧动画 | **简化版** | 减少笔触抖动，动画更平滑 |
| 打算用Spine骨骼动画 | **简化版** | 色块清晰，方便分层和绑定 |
| 做静态展示（如解锁界面大图）| **完整版** | 细节丰富，更精致 |

### 如果资源有限，优先顺序：

**先做简化版（动画核心）**：简化版图9-14（宠物表情） → 简化版动画1-6（idle帧）
**再做完整版（视觉包装）**：批次1-2（Logo、背景、UI） → 批次4-5（背景、碗、食物）

简化版出完后，宠物动画就能跑起来了。完整版可以慢慢补。
