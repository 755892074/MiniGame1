## 2. 模块职责

### 2.1 共享核心层

| 类 | 职责 | 生命周期 |
|---|------|---------|
| `GameManager` | 游戏全局状态、场景切换、暂停/恢复 | 全局单例 |
| `AdManager` | 广告SDK初始化、加载、展示、回调 | 全局单例 |
| `DataStorage` | 玩家数据（金币、最高分、进度）读写 | 全局单例 |
| `SceneLoader` | 异步场景加载 + 过渡动画 | 全局单例 |
| `EventCenter` | 事件发布订阅，模块解耦 | 全局单例 |

### 2.2 找茬大师模块

| 类 | 职责 |
|---|------|
| `FindDiffController` | 游戏主控制器，协调各子系统 |
| `DifferenceManager` | 差异点加载、点击判定、进度统计 |
| `LevelData` (SO) | 关卡数据 ScriptableObject |
| `LevelSelectUI` | 章节选择界面 |
| `GameUI` | HUD（计时器、分数、进度） |
| `ResultUI` | 结算界面（星级、奖励、广告） |
| `HintSystem` | 提示道具逻辑 |

### 2.3 无厘头大挑战模块

| 类 | 职责 |
|---|------|
| `QuizController` | 答题主控制器，流程管理 |
| `QuestionBank` (SO) | 题库 ScriptableObject |
| `QuizUI` | 题目/选项展示界面 |
| `ComboSystem` | 连击计数 + 倍率计算 |
| `LifeSystem` | 生命值管理 + 复活逻辑 |
| `ResultUI` | 结算 + 分享界面 |

## 3. 数据流

```
┌─────────────┐    Event    ┌─────────────┐
│   UI 层      │←──────────→│  逻辑层       │
│ (Canvas)    │             │ (Controller) │
└──────┬──────┘             └──────┬───────┘
       │                          │
       │ UnityEvent/              │ 读写
       │ Button.onClick           ↓
       │                ┌─────────────┐
       └───────────────→│  数据层      │
                        │ (SO/JSON)   │
                        └─────────────┘
```

**事件驱动通信**：
- UI 按钮 → Controller 方法调用（直接引用）
- Controller → UI 更新（通过 EventCenter 事件）
- Controller → AdManager（通过 EventCenter 事件）
- 游戏状态变化 → 所有监听者（通过 FSM 状态切换）

## 4. 游戏状态机

```
            ┌─────────┐
            │  INIT   │ 应用启动
            └────┬────┘
                 ↓
            ┌─────────┐
     ┌──────│  MENU   │ 主菜单
     │      └────┬────┘
     │           ↓
     │      ┌─────────┐
     │      │ PLAYING │ 游戏中
     │      └────┬────┘
     │           ↓
     │      ┌─────────┐
     │  ┌───│ PAUSED  │ 暂停
     │  │   └────┬────┘
     │  │        ↓
     │  └──── PLAYING (恢复)
     │           ↓
     │      ┌─────────┐
     │      │ RESULT  │ 结算
     │      └────┬────┘
     │           ↓
     └─────── MENU (返回)
```

使用 F8Framework FSM 模块实现状态转换。

## 5. 编辑器工具计划

为提高 AI 协作效率，通过 Codely 构建编辑器工具：

| 工具 | 用途 | 方式 |
|------|------|------|
| 关卡生成器 | 批量生成 `LevelData` SO | Codely execute_csharp_script |
| 题库编辑器 | 可视化编辑 `QuestionBank` | 编辑器窗口 (EditorWindow) |
| 差异点标记工具 | 在图片上标记差异区域 | 编辑器工具 |
| 批量导入工具 | AI 生图后批量导入+命名 | Codely unity_asset 操作 |
