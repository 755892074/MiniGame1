# 疯狂铲屎官 — AI 美术工作流速查卡

## 完整工作流（3步出图 → 自动部署）

```
第1步：生成图片
  ├─ 打开 prompts/ 目录
  ├─ 找到对应类别的 .md 文件
  ├─ 复制提示词 → 粘贴到 AI 绘图工具
  └─ 保存图片到 _raw/ 目录

第2步：批量处理
  ├─ 如果生成的是 Sprite Sheet（多帧在一张图）：
  │   └─ python scripts/spritesheet_cutter.py <图> --cols 4 --rows 1 --prefix "cat_idle_s2_frame"
  ├─ 运行处理脚本：
  │   └─ python scripts/batch_process.py
  └─ 自动完成：去白底 → 裁切 → 缩放 → 输出到 _processed/

第3步：自动部署
  ├─ 先预览（不实际复制）：
  │   └─ python scripts/auto_deploy.py --dry-run
  ├─ 确认无误后实际部署：
  │   └─ python scripts/auto_deploy.py
  └─ 图片自动复制到 Unity 项目 Assets/ 对应路径
```

## 文件命名速查

### 帧动画命名（最重要！脚本靠文件名识别）

```
{pet}_{action}_stage{N}_frame{XX}.png

pet:     cat / dog / hamster / parrot / fish / rabbit
action:  idle / happy / cry / run / cute
stage:   1(脏) / 2(干净) / 3(幸福)
frame:   01 / 02 / 03 / 04 (或 05/06 给run)

示例：
  cat_idle_stage2_frame01.png     ← 小橘 待机 干净版 第1帧
  dog_happy_stage3_frame03.png    ← 阿黄 开心 幸福版 第3帧
  hamster_cry_stage1_frame02.png  ← 球球 哭泣 脏版 第2帧
```

### 其他资源命名

```
house_lv1.png ~ house_lv5.png       ← 住所
foodbowl_lv1.png                    ← 食盆区
toy_lv1.png                         ← 玩具区
medical_lv1.png                     ← 医疗区
garden_lv1.png                      ← 花园
yard_bg_lv1.png ~ lv5.png           ← 小院背景
chapter_01.png ~ chapter_05.png     ← 章节封面
icon_fish.png / icon_medal.png      ← 货币图标

```

## 快速命令

```bash
# 切Sprite Sheet（4帧横排）
python scripts/spritesheet_cutter.py "小橘_idle.png" --cols 4 --rows 1 --prefix "cat_idle_s2_frame"

# 切Sprite Sheet（6帧横排，run动画）
python scripts/spritesheet_cutter.py "小橘_run.png" --cols 6 --rows 1 --prefix "cat_run_s2_frame"

# 批量处理_raw目录下所有图片
python scripts/batch_process.py

# 预览部署（不实际复制）
python scripts/auto_deploy.py --dry-run

# 实际部署到项目
python scripts/auto_deploy.py

# 指定其他项目路径
python scripts/auto_deploy.py "D:/MyOtherProject"
```

## 常见问题

**Q: AI生成的图背景不是纯白怎么办？**
A: batch_process.py 的阈值是 240（RGB值≥240视为白底），如果你的背景偏灰，可以修改脚本中的 `threshold=240` 调低。

**Q: 生成的图尺寸不对怎么办？**
A: 不用担心，脚本会自动裁切到非透明内容边界，再缩放到目标尺寸（帧动画256×256，图标128×128）。

**Q: 想覆盖已有资源怎么办？**
A: 直接重新生成 → 处理 → 部署即可，auto_deploy.py 默认覆盖同名文件。

**Q: 金鱼没有run动画怎么办？**
A: 用 swim 替代：`fish_swim_stage2_frame01.png`，脚本里的 ACTION_MAP 可以扩展。

**Q: 一次生成太多帧，怎么管理？**
A: 按宠物+动作+阶段分包生成，每完成一个组合就跑一次 batch_process，不要攒太多。
