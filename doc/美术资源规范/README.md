# 疯狂铲屎官 — 美术资源规范总纲

> 本目录包含所有美术资源的命名规范、尺寸要求、AI生成提示词和自动化工具。
> 用法：按 `prompts/` 里的提示词去 AI 生成 → 产出的图丢到 `_raw/` → 跑 `scripts/` 里的脚本自动切图、命名、放到项目对应路径。

## 目录结构

```
美术资源规范/

│   ├── 01_宠物帧动画.md            ← 6只宠物 × 5动作 × 3阶段

│   ├── 04_UI图标与按钮.md          ← 货币/称号/道具图标
│   └── 05_音效说明.md             ← 音效需求（非AI生成）
├── scripts/                       ← 自动化工具
│   ├── spritesheet_cutter.py      ← 切 Sprite Sheet
│   ├── auto_deploy.py             ← 自动命名+部署到项目
│   └── batch_process.py           ← 批量处理（去背景+裁切+重命名）
└── _raw/                          ← AI生成的原始图片（临时目录）
    └── (把AI生成的图丢这里)
```

## 全局风格锚点

所有美术资源必须遵守以下风格规则，保持视觉一致性：

| 属性 | 规范 |
|---|---|
| **画风** | Q版扁平 (flat design / chibi) |
| **线条** | 粗线条描边 (thick outline, 2-3px) |
| **配色** | 明亮柔和色调 (pastel color palette) |
| **渲染** | 纯平涂，无渐变/无光影 |
| **背景** | 纯白背景 `#FFFFFF`（便于自动去背景） |
| **帧动画尺寸** | 单帧 128×128 px（小游戏包体优化） |
| **Sprite Sheet** | 每只宠物一张 512×512 px（统一压缩） |
| **图标尺寸** | 96×96 px（UI图标） |
| **场景尺寸** | 750×1334 px（竖屏全屏，CDN加载） |
| **帧率** | 8-10 fps |
| **格式** | PNG（帧动画/图标）/ WebP（场景大图） |

## 小游戏包体限制与资源分层策略

> **核心约束：首包 ≤ 4MB（微信/抖音统一），美术资源必须分层管理。**

### 三层资源架构

| 层级 | 位置 | 大小限制 | 放什么 | 加载方式 |
|---|---|---|---|---|
| **第1层 首包** | 主包内 Resources/ | ≤ 4MB（含引擎代码） | 核心UI预制体、食物/碗图标、前2关数据、货币图标 | 随包下载 |
| **第2层 分包** | 小游戏分包 | 单包≤20MB，总量≤20MB | 宠物帧动画sprite sheet、住所图标、关卡数据 | 玩家进入功能时按需下载 |
| **第3层 CDN** | 远程服务器 | 无限制 | 场景背景大图、章节封面、音效/BGM | AutoStreaming自动流式加载 |

### 团结引擎 AutoStreaming

团结引擎 2022.3 自带 **AutoStreaming** 工具，自动处理：
- 打包时把大图/音频上传到 UOS CDN
- 运行时按需流式加载，无需手写加载逻辑
- 首包只保留代码和小图标

**用法：** Unity菜单 → AutoStreaming → 配置 Texture/Scene/Audio Streaming → 打包时自动上传

### 各资源的归属层级

| 资源 | 层级 | 理由 |
|---|---|---|
| 食物图标（15张） | 首包 | 关卡必需，单张约15KB |
| 碗图标（16张） | 首包 | 关卡必需 |
| UI按钮/货币图标 | 首包 | 小图标，总量<200KB |
| 宠物帧动画（240帧） | 分包 | 量大但可按宠物分包 |
| 住所/建筑图标（13张） | 分包 | 小院功能才需要 |
| 场景背景（5张大图） | CDN | 单张>500KB，AutoStreaming |
| 章节封面（5张） | CDN | 过场才显示 |
| 音效/BGM（15个） | CDN | AutoStreaming Audio |

### 帧动画尺寸优化

**为什么用 128×128 而不是 256×256？**

| 尺寸 | 单帧PNG | 240帧总量 | 适合 |
|---|---|---|---|
| 256×256 | ~30KB | ~7.2MB | ❌ 分包装不下 |
| 128×128 | ~8KB | ~1.9MB | ✅ 单个分包内 |
| 96×96 | ~4KB | ~0.96MB | ✅ 极致优化 |

128×128 在手机屏幕上放大2倍（256px显示区域）仍然清晰，休闲游戏完全够用。

### Sprite Sheet 合并规则

每只宠物所有帧动画合并为一张 sprite sheet：

```
布局：512×512 px
内含：5动作 × 3阶段 的所有帧，按行排列
  行1: idle_s1 (4帧) | idle_s2 (4帧) | idle_s3 (4帧)
  行2: happy_s2 (4帧) | happy_s3 (4帧) | cry_s1 (4帧)
  行3: run_s2 (6帧) | run_s3 (6帧)
  行4: cute_s2 (2帧) | cute_s3 (2帧) | 空白

每帧 128×128，4×4=16列 → 512px宽
4行 → 512px高
```

**6只宠物 = 6张 512×512 sprite sheet ≈ 1.8MB（可放入一个分包）**

## 制作流程（3步出图）

```
第1步：生成
  打开 prompts/ 目录下对应的 .md 文件
  复制提示词 → 粘贴到 AI 绘图工具（推荐 Midjourney / DALL-E / Stable Diffusion）
  把生成的图保存到 _raw/ 目录

第2步：处理
  运行 scripts/batch_process.py
  自动：去白底 → 裁切到目标尺寸 → 输出到 _processed/

第3步：部署
  运行 scripts/auto_deploy.py
  自动：按命名规则重命名 → 复制到项目 Assets/ 对应路径
  如果同名文件已存在则覆盖
```
